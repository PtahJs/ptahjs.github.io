Developing Plugins ​

Plugins are used to extend the functionality of the engine, adding additional capabilities or features.

Plugin Format ​

Plugins need to adhere to a specific format. Here is an example format for a plugin:

js
export default {
  name: "PluginName",
  install(engine) {
    let name = "";

    // Define a method to set the name
    engine.$setName = (val) => {
      name = val;
    };

    // Define a method to get the name
    engine.$getName = () => {
      return name;
    };
  },
};

In this example, the install method is used to inject the plugin's functionality into the engine instance. The plugin provides two new methods: $setName and $getName, for setting and getting the name value.

Installing Plugins ​

To use a plugin, you first need to install it into the engine instance. Here’s how to install a plugin:

js
import TestPlugin from "./TestPlugin";
import { createPtah } from "@ptahjs/core";

const ptah = createPtah({
  id: "ve1",
});

// Install the plugin
ptah.use(TestPlugin);

In this example, the createPtah function is used to create an engine instance, and then the plugin is installed using the ptah.use method.

Using Plugin Features ​

Once the plugin is installed, you can use the features provided by the plugin in the engine instance. For example, in a .vue file, you can use the plugin’s features like this:

js
import { useApp } from "@ptahjs/core";

const app = useApp();

// Use the plugin's $setName method
app.$setName("Test");

// Use the plugin's $getName method
console.log(app.$getName()); // Output: 'Test'

In this example, the useApp hook is used to get the application instance, and then you can use the methods provided by the plugin to manipulate data or retrieve results.

Plugins make it easy to extend the functionality of the engine, making it more flexible and powerful.
